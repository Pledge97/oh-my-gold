# 黄金积存金做T量化策略设计文档

**日期：** 2026-05-14
**版本：** v1.0
**策略名称：** 双模式自适应做T策略

---

## 一、策略概述

### 核心思想

根据市场状态自动切换两种交易模式：
- **震荡模式：** 用布林带均值回归逻辑，低买高卖
- **趋势模式：** 用顺势回调逻辑，跟踪趋势让利润奔跑

### 适用场景

- 银行积存金 T+0 交易
- 数据更新频率：5秒
- 时间维度：主做日内，趋势明确时允许隔夜

### 资金配置

| 仓位 | 数量 | 说明 |
|------|------|------|
| 底仓 | 50g  | 长期持有，不参与做T |
| T仓  | 100g | 做T主力仓位 |
| 现金 | 50g  | 备用资金，补仓/机动 |

---

## 二、交易成本

- **买入手续费：** 0%
- **卖出手续费：** 0.4%
- **最低盈利目标：** 每笔卖出盈利 ≥ 0.6%（覆盖0.4%手续费 + 0.2%净利润）

---

## 三、市场状态识别

### 指标

- **ADX(14)：** 基于日线K线计算，判断趋势强度
- **+DI / -DI：** 判断趋势方向

### 状态分类

| 条件 | 状态 | 激活策略 |
|------|------|---------|
| ADX < 25 | 震荡市 | 布林带均值回归 |
| ADX ≥ 25，且 ADX_t > ADX_{t-3}（趋势上升），+DI > -DI | 上涨趋势形成中 | 趋势做多T |
| ADX ≥ 25，且 ADX_t > ADX_{t-3}（趋势上升），-DI > +DI | 下跌趋势形成中 | 暂停买入，只平仓 |
| ADX ≥ 25，但 ADX_t ≤ ADX_{t-3}（趋势衰减） | 趋势衰减中 | 降级为震荡模式，不开新仓 |

> **说明：** ADX_{t-3} 表示3个日线周期前的ADX值。ADX上升说明趋势正在增强，ADX下降即使仍高于25也说明趋势动能在衰减，此时按震荡模式保守处理，持仓中的趋势仓也应收紧止盈条件（盈利回撤0.5%即平仓，而非等EMA跌破）。

---

## 四、震荡模式交易规则

### 指标参数

| 指标 | 参数 | K线周期 |
|------|------|---------|
| 布林带 | 周期20，标准差2.0 | 5分钟 |
| RSI | 周期14 | 5分钟 |
| ATR | 周期14 | 5分钟 |

### 买入信号

满足以下全部条件时买入：
1. 日线 ADX < 25（震荡市确认）
2. 1分钟布林带：价格触及或跌破下轨
3. 买入数量：T仓的20%（20g）

### 卖出信号

满足任意一条时卖出：
1. 价格 ≥ 买入价 × 1.006（盈利 0.6%）
2. 价格触及1分钟布林上轨且盈利 ≥ 0.4%

### 止损

满足任意一条时止损平仓：
1. 价格 ≤ 买入价 - 2×ATR₁₄（动态止损，随波动率自动调整）
2. 持仓超过1个交易日且趋势转为下跌（ADX ≥ 25，-DI > +DI）

### 加仓规则

- 价格每下跌超过 1.5×ATR₁₄ 触发一次加仓
- 加仓递减：第1次加20g，第2次加10g，第3次加5g
- 同一信号最多加仓 3 次（最大持仓 20+20+10+5 = 55g）

---

## 五、趋势模式交易规则

### 指标参数

| 指标 | 参数 | K线周期 |
|------|------|---------|
| EMA | 20, 60 | 4小时 |
| EMA | 20 | 5分钟 |
| RSI | 14 | 1分钟 |

### 上涨趋势（ADX ≥ 25，+DI > -DI）

**买入信号**（满足全部条件）：
1. 4小时 EMA20 > EMA60（中期上涨确认）
2. 1分钟 RSI(14) 回调至 ≤ 40（短期超卖）
3. 买入数量：20g

**卖出信号（趋势跟踪止盈）**：
1. 价格跌破5分钟 EMA20，且当前盈利 ≥ 0.8%
2. 允许隔夜持仓，直到趋势结束（EMA跌破为信号）

**止损**：
1. 价格 ≤ 买入价 - 2×ATR₁₄（动态止损）
2. 或亏损达到固定 -1.5%（兜底保护）

### 下跌趋势（ADX ≥ 25，-DI > +DI）

- 暂停所有买入操作
- 对已持仓执行正常止盈/止损规则
- 等待 ADX 回落至 25 以下或趋势反转（+DI 反超 -DI）后恢复操作

---

## 六、仓位管理

### 单次开仓规模

- 每次买入：20g（T仓的20%）
- 最大同时持仓：5个独立仓位（合计100g）

### 现金储备使用规则

- T仓100g全部买入后，如仍有强烈信号，可动用现金仓（最多30g）
- 优先回补现金储备，确保备用资金充足

---

## 七、风控规则

### 单笔止损

| 条件 | 动作 |
|------|------|
| 亏损 ≥ -1.5% | 强制平仓止损 |

### 日内总风控

| 条件 | 动作 |
|------|------|
| 当日累计亏损 ≥ T仓的3%（3g价值） | 暂停交易至次日 |
| 连续3日亏损 | 降低单次开仓至10g，观察3日后恢复 |

### 隔夜持仓条件

| 条件 | 说明 |
|------|------|
| 仅在趋势模式下允许隔夜 | ADX ≥ 25，+DI > -DI |
| 当前持仓盈利 ≥ 0.5% | 已盈利才允许隔夜 |
| 隔夜保护止损 | 从最高价回撤 0.8% 时平仓 |

---

## 八、熔断机制

熔断分三级，触发后系统暂停开新仓，已持仓按正常止盈/止损执行。

### 一级熔断：价格异常波动（短期）

| 触发条件 | 暂停时长 | 恢复条件 |
|---------|---------|---------|
| 5秒内价格变动 ≥ 0.5% | 暂停开仓 10 分钟 | 10分钟后自动恢复 |
| 5分钟内价格变动 ≥ 1.5% | 暂停开仓 30 分钟 | 30分钟后自动恢复 |

> 触发原因：突发新闻、黑天鹅事件导致短期价格跳动，此时布林带/RSI信号不可靠。

---

### 二级熔断：波动率异常飙升

| 触发条件 | 暂停时长 | 恢复条件 |
|---------|---------|---------|
| 当前 ATR₁₄ ≥ 历史ATR₁₄均值的 3 倍 | 暂停开仓至当日收盘 | 次日 ATR 恢复正常后解除 |

> 触发原因：极端市场环境下止损幅度会大幅扩大，动态止损可能导致单笔亏损超出预期上限。

---

### 三级熔断：连续止损

| 触发条件 | 暂停时长 | 恢复条件 |
|---------|---------|---------|
| 同一交易日内触发止损 ≥ 3 次 | 暂停至次日 | 次日开盘人工确认后恢复 |
| 连续 3 个交易日每日触发止损 ≥ 2 次 | 暂停 3 个交易日 | 3日后人工审查策略参数后恢复 |

> 触发原因：连续止损说明市场环境与策略假设严重不符，需要人工介入评估。

---

### 熔断状态与信号流程的关系

```
每次5秒更新时，先检查熔断状态：
    │
    ├─ 熔断激活中？
    │       ├─ 是 → 跳过开仓逻辑，仅执行已持仓的止盈/止损检查
    │       └─ 否 → 正常执行策略信号逻辑
    │
    └─ 执行完毕后，检查是否触发新的熔断条件
```

---

### 熔断日志

每次熔断触发须记录：
- 触发时间
- 触发级别（一/二/三级）
- 触发条件具体数值
- 恢复时间
- 熔断期间持仓盈亏

## 九、多周期数据架构

| K线周期 | 用途 |
|---------|------|
| 5秒 | 实时价格更新，触发信号检测，一级熔断检测 |
| 5分钟 | 布林带、RSI、ATR计算，趋势跟踪止盈（EMA20） |
| 4小时 | 中期趋势确认（EMA20/60） |
| 日线 | 大趋势判断（ADX_14），二级熔断ATR基准计算 |

## 十、信号流程

```
5秒数据更新
    │
    ├─ 【熔断检查】当前是否处于熔断状态？
    │       ├─ 是 → 跳过开仓，仅检查持仓止盈/止损 → 结束
    │       └─ 否 → 继续
    │
    ├─ 合成5分钟/4小时/日线K线
    │
    ├─ 计算日线ADX → 判断市场状态
    │       │
    │       ├─ 震荡模式（ADX < 25，或 ADX ≥ 25 但趋势衰减 ADX_t ≤ ADX_{t-3}）
    │       │       ├─ 计算5分钟布林带、RSI、ATR
    │       │       ├─ 判断买入/卖出信号
    │       │       └─ 执行加仓/止损逻辑
    │       │
    │       └─ 趋势模式（ADX ≥ 25 且 ADX_t > ADX_{t-3}）
    │               ├─ 判断趋势方向（+DI vs -DI）
    │               ├─ 上涨：检测RSI回调买入信号
    │               ├─ 上涨：检测5分钟EMA20跌破止盈
    │               └─ 下跌：暂停买入，执行平仓
    │
    ├─ 检查风控规则（日内亏损上限、单笔止损）
    │
    ├─ 【熔断触发检查】是否满足一/二/三级熔断条件？
    │       └─ 是 → 记录熔断日志，激活对应级别熔断
    │
    └─ 生成信号记录（买入/卖出/持仓/止损/暂停/熔断）
```

## 十一、绩效指标

系统需记录以下数据用于策略评估：

| 指标 | 说明 |
|------|------|
| 总收益（g） | 以黄金克数计算净盈亏 |
| 总收益（元） | 以人民币计算净盈亏 |
| 胜率 | 盈利笔数 / 总笔数 |
| 盈亏比 | 平均盈利 / 平均亏损 |
| 最大回撤 | 总资产从峰值的最大下跌幅度 |
| 日均交易次数 | 评估策略活跃度 |
| 手续费总支出 | 累计卖出手续费成本 |
| 夏普比率 | 风险调整后收益 |

---

## 十二、待优化项

1. **参数优化：** 布林带周期、RSI阈值、ADX切换阈值均需通过历史数据回测验证
2. **滑点处理：** 银行积存金无买卖价差，交易成本固定为卖出0.4%手续费，已纳入策略计算
3. **交易时段：** 工作日24小时连续交易，夜盘波动较大，熔断机制在夜盘同样生效
4. **趋势反转信号：** 当前依赖ADX被动识别，未来可引入更灵敏的趋势反转指标
