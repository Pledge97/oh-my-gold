# 黄金积存金做T量化策略设计文档

**日期：** 2026-05-14
**版本：** v2.0
**策略名称：** 双模式自适应做T策略（组合仓位管理版）

---

## 一、策略概述

### 核心思想

根据市场状态自动切换两种交易模式：
- **震荡模式：** 用布林带均值回归逻辑，分批建仓，组合止盈/止损
- **趋势模式：** 用顺势回调逻辑，跟踪趋势让利润奔跑

### 适用场景

- 银行积存金 T+0 交易
- 数据更新频率：5秒
- 时间维度：主做日内，趋势明确时允许隔夜

### 资金配置

| 仓位 | 数量 | 说明 |
|------|------|------|
| 底仓 | 50g  | 长期持有，不参与做T |
| T仓  | 100g | 做T主力仓位（分批建仓） |
| 现金 | 50g  | 备用资金，机动 |

---

## 二、交易成本

- **买入手续费：** 0%
- **卖出手续费：** 0.4%
- **最低盈利目标：** T仓整体盈利 ≥ 0.6%（覆盖手续费后有净利润）

---

## 三、市场状态识别

### 指标

- **ADX(14)：** 基于日线K线计算，判断趋势强度
- **+DI / -DI：** 判断趋势方向

### 状态分类

| 条件 | 状态 | 激活策略 |
|------|------|---------|
| ADX < 25 | 震荡市 | 布林带均值回归 |
| ADX ≥ 25，且 ADX_t > ADX_{t-3}（趋势上升），+DI > -DI | 上涨趋势形成中 | 趋势做多T |
| ADX ≥ 25，且 ADX_t > ADX_{t-3}（趋势上升），-DI > +DI | 下跌趋势形成中 | 暂停买入，触发清仓 |
| ADX ≥ 25，但 ADX_t ≤ ADX_{t-3}（趋势衰减） | 趋势衰减中 | 降级为震荡模式，不开新仓 |

> **说明：** ADX_{t-3} 表示3个日线周期前的ADX值。趋势衰减时收紧止盈条件（盈利回撤0.5%即平仓）。

---

## 四、分批建仓规则

T仓采用**组合仓位管理**，将多次买入视为一个整体，统一计算浮盈浮亏。

### 建仓计划

| 批次 | 触发条件 | 买入数量 | 累计持仓 |
|------|---------|---------|---------|
| 第1批（初始开仓） | 价格触及布林下轨 | 50g | 50g |
| 第2批（加仓） | 价格再跌 1×ATR₁₄ | 30g | 80g |
| 第3批（加仓） | 价格再跌 1×ATR₁₄ | 20g | 100g（满仓） |

> **加仓前提：** T仓整体浮亏 < -1.5%，且未触发波动熔断

### 加仓禁止条件

| 条件 | 动作 |
|------|------|
| T仓整体浮亏 ≥ -1.5% | 停止加仓，等待反弹 |
| ATR₁₄ ≥ 历史均值 × 1.8 | 波动熔断，禁止新仓 |
| 趋势转空（-DI > +DI 且 ADX 上升） | 禁止加仓，执行清仓 |

---

## 五、组合止盈规则

止盈以 **T仓整体盈利率** 为基准（整体盈利 = 当前价 × 总持仓量 - 总成本 - 手续费）。

### 分批止盈计划

| 批次 | 触发条件 | 卖出数量 | 说明 |
|------|---------|---------|------|
| 第1次止盈 | T仓整体盈利 ≥ 0.6% | 卖出T仓持仓的 60% | 锁定大部分利润 |
| 第2次止盈 | T仓整体盈利 ≥ 1.2% | 卖出T仓持仓的 20% | 继续持有剩余20%追涨 |
| 第3次止盈 | 金价跌破5分钟EMA20 | 卖出T仓全部剩余持仓 | 趋势结束信号，清空 |

> **说明：** 每次止盈后重新计算持仓量和成本，下次止盈基于新的持仓状态。

---

## 六、组合止损规则

止损以 **T仓整体浮亏率** 为基准，不对单笔持仓单独止损。

### 分级止损计划

| 级别 | 触发条件 | 动作 |
|------|---------|------|
| 停止加仓 | T仓整体浮亏 ≥ -1.5% | 禁止新建仓位，等待反弹 |
| 强制减半 | T仓整体浮亏 ≥ -2.5% | 立即卖出T仓持仓的 50% |
| 清仓 | T仓整体浮亏 ≥ -3.5% | 立即清空T仓全部持仓 |
| 趋势清仓 | 趋势转空（-DI > +DI 且 ADX 上升） | 立即清空T仓全部持仓 |

> **整体浮亏率计算：**
> `浮亏率 = (当前价 × 总持仓量 - 总成本) / 总成本`
> 其中总成本 = Σ(各批次买入价 × 买入量)

---

## 七、趋势模式交易规则

### 指标参数

| 指标 | 参数 | K线周期 |
|------|------|---------|
| EMA | 20, 60 | 4小时 |
| EMA | 20 | 5分钟 |
| RSI | 14 | 5分钟 |

### 上涨趋势（ADX ≥ 25，+DI > -DI）

**买入信号**（满足全部条件）：
1. 4小时 EMA20 > EMA60（中期上涨确认）
2. 5分钟 RSI(14) 回调至 ≤ 40（短期超卖）
3. 初始买入 50g，后续加仓规则同震荡模式

**止盈信号（趋势跟踪）**：
1. 第1次止盈：T仓盈利 ≥ 0.6%，卖出60%
2. 第2次止盈：T仓盈利 ≥ 1.2%，卖出20%
3. 第3次止盈：金价跌破5分钟EMA20，清空剩余

**止损**：同组合止损规则（-1.5% 停加仓，-2.5% 减半，-3.5% 清仓）

### 下跌趋势（ADX ≥ 25，-DI > +DI）

- 暂停所有买入操作
- 立即触发趋势清仓，清空T仓全部持仓
- 等待 ADX 回落至 25 以下或趋势反转（+DI 反超 -DI）后恢复操作

---

## 八、风控规则

### 日内总风控

| 条件 | 动作 |
|------|------|
| 当日累计亏损 ≥ T仓总价值的3% | 暂停交易至次日 |
| 连续3日亏损 | 降低初始开仓至25g，观察3日后恢复 |

### 隔夜持仓条件

| 条件 | 说明 |
|------|------|
| 仅在趋势模式下允许隔夜 | ADX ≥ 25，+DI > -DI |
| T仓整体盈利 ≥ 0.5% | 已盈利才允许隔夜 |
| 隔夜保护止损 | T仓从最高盈利回撤 0.8% 时清仓 |

---

## 九、熔断机制

熔断分三级，触发后系统暂停开新仓，已持仓按正常止盈/止损执行。

### 一级熔断：价格异常波动（短期）

| 触发条件 | 暂停时长 | 恢复条件 |
|---------|---------|---------|
| 5秒内价格变动 ≥ 0.5% | 暂停开仓 10 分钟 | 10分钟后自动恢复 |
| 5分钟内价格变动 ≥ 1.5% | 暂停开仓 30 分钟 | 30分钟后自动恢复 |

### 二级熔断：波动率异常飙升

| 触发条件 | 暂停时长 | 恢复条件 |
|---------|---------|---------|
| 当前 ATR₁₄ ≥ 历史ATR₁₄均值的 1.8 倍 | 禁止新建仓位 | ATR 恢复正常后解除 |

> 注：二级熔断阈值从3倍调整为1.8倍，更早介入保护。

### 三级熔断：连续止损

| 触发条件 | 暂停时长 | 恢复条件 |
|---------|---------|---------|
| 同一交易日内触发清仓 ≥ 2 次 | 暂停至次日 | 次日开盘人工确认后恢复 |
| 连续 3 个交易日每日触发清仓 | 暂停 3 个交易日 | 3日后人工审查策略参数后恢复 |

## 十、多周期数据架构

| K线周期 | 用途 |
|---------|------|
| 5秒 | 实时价格更新，触发信号检测，一级熔断检测 |
| 5分钟 | 布林带、RSI、ATR计算，趋势跟踪止盈（EMA20） |
| 4小时 | 中期趋势确认（EMA20/60） |
| 日线 | 大趋势判断（ADX_14），二级熔断ATR基准计算 |

---

## 十一、信号流程

```
5秒数据更新
    │
    ├─ 【熔断检查】当前是否处于熔断状态？
    │       ├─ 是 → 跳过开仓，仅检查持仓止盈/止损 → 结束
    │       └─ 否 → 继续
    │
    ├─ 合成5分钟/4小时/日线K线
    │
    ├─ 计算日线ADX → 判断市场状态
    │
    ├─ 计算T仓整体浮盈浮亏率
    │       │
    │       ├─ 浮亏 ≥ -3.5% 或趋势转空 → 清仓
    │       ├─ 浮亏 ≥ -2.5% → 强制减半
    │       └─ 浮亏 ≥ -1.5% → 停止加仓
    │
    ├─ 检查止盈条件
    │       ├─ 盈利 ≥ 0.6% → 卖出60%
    │       ├─ 盈利 ≥ 1.2% → 卖出20%
    │       └─ 金价跌破5分钟EMA20 → 清空剩余
    │
    ├─ 检查建仓/加仓条件（浮亏 < -1.5% 且无熔断）
    │       ├─ 无持仓 + 价格触及布林下轨 → 初始开仓50g
    │       ├─ 持仓80g以下 + 价格再跌1×ATR → 加仓
    │       └─ 持仓已满100g → 不加仓
    │
    ├─ 【熔断触发检查】是否满足熔断条件？
    │       └─ 是 → 记录熔断日志，激活对应级别熔断
    │
    └─ 生成信号记录（买入/卖出/加仓/止盈/止损/熔断）
```

## 十二、绩效指标

系统需记录以下数据用于策略评估：

| 指标 | 说明 |
|------|------|
| 总收益（g） | 以黄金克数计算净盈亏 |
| 总收益（元） | 以人民币计算净盈亏 |
| 胜率 | 盈利轮次 / 总轮次（一轮 = 从建仓到清仓） |
| 盈亏比 | 平均盈利 / 平均亏损 |
| 最大回撤 | T仓从峰值的最大下跌幅度 |
| 日均建仓次数 | 评估策略活跃度 |
| 手续费总支出 | 累计卖出手续费成本 |
| 平均持仓时长 | 从初始建仓到清仓的平均时间 |

## 十三、待优化项

1. **参数优化：** 布林带周期、ATR加仓间距、止盈/止损阈值均需通过历史数据回测验证
2. **交易成本：** 银行积存金无买卖价差，交易成本固定为卖出0.4%手续费，已纳入策略计算
3. **交易时段：** 工作日24小时连续交易，夜盘波动较大，熔断机制在夜盘同样生效
4. **趋势反转信号：** 当前依赖ADX被动识别，未来可引入更灵敏的趋势反转指标
5. **回测验证：** 新的组合仓位管理规则需要用历史tick数据回测，验证参数合理性
